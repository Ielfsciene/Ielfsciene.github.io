Evolution of a Notification Product

Dates: 11/22 to 01/23, 02/23

Project Overview: Enhanced an existing WPF-based notification application by adding rich text functionalities to an existing window. Specifically, added support for bold, italic, and underlined text, as well as hyperlinks and images.

Your Role: Sole developer responsible for implementing the new features. Collaborated with the QA team for validation and a senior coworker for guidance.

Technologies and Tools Used: WPF, C#

Key Challenges and Solutions: The primary challenge was ensuring backwards compatibility between the previous version, which stored notification details as plain text, and the new version, which stored the full contents of the FlowDocument of the rich text box. Addressed this by either stripping all tags from the text or creating and adding tags to make the text compatible.

Outcomes and Impact: The new version allowed the company to link to documentation and include images to communicate information more effectively. Images were stored as base64 in the database to keep them lightweight.

D365FO Developments (Capstone Project)

Dates: 03/23 to 04/24

Project Overview: The primary objective was to perform lifecycle maintenance on our existing ERP software through Microsoft Dynamics 365 Finance & Operations (D365FO).

Your Role: One of three developers on the team, working closely with a dedicated QA tester for Finance & Operations. Initially handled simple tasks, then progressed to essential aspects of the system such as data entities, menu items, forms, run dialogs, and localization through label files.

Technologies and Tools Used: Microsoft Dynamics 365 Finance & Operations, X++

Key Challenges and Solutions: The niche nature of FO technology means limited public resources, relying heavily on Microsoft documentation, which can be complex. Often, the best solutions were found through collaboration with colleagues.

Major Phases and Activities:

  • Training (4 weeks): Comprehensive training on FO basics.
  • Data Entities: Managed and created data entities.
  • Code Execution: Developed run dialogs and other code components.
  • Menu Items and Forms: Developed and maintained essential UI components.
  • Localization: Managed localization using label files.
  • API Usage: Utilized the FO API through ODATA and Postman.

Outcomes and Impact: Contributed to maintaining and enhancing the ERP system, ensuring smooth operations and updated functionalities. The cumulative effort of microprojects formed the basis of significant system improvements, although specific metrics are confidential.

Azure Active Directory Management Tool

Dates: 06/23 to 07/23

Project Overview: Evolved an existing small tool for the infrastructure team to manage Azure Active Directory (AAD). Added new functionality to allow deactivating inactive directories and deleting orphaned directories.

Your Role: Sole developer responsible for implementing the new features. Worked directly with the infrastructure team, who provided requirements and conducted testing.

Technologies and Tools Used: .NET, C#

Key Challenges and Solutions: The main challenge was verifying the functionality of the code since I did not have the necessary permissions to alter the AAD and there was no testing environment available. Collaborated closely with the infrastructure team for validation and feedback.

Outcomes and Impact: Successfully enhanced the tool to improve the infrastructure team's ability to manage AAD. This allowed for better maintenance by efficiently deactivating inactive directories and removing orphaned directories, thereby streamlining the management process.

Functional Analysis for Product Evolution

Dates: 02/23

Project Overview: Conducted a functional analysis to estimate how the existing product could be further adapted to serve the entire group of companies, rather than just a few services within the company.

Your Role: Sole author responsible for conducting the functional analysis. Collaborated with my manager and the Product Owner (PO) for reviews and feedback.

Technologies and Tools Used: Utilized an existing functional analysis (FA) template and a completed FA as references. No specific methodologies or frameworks were involved.

Key Challenges and Solutions: The main challenge was presenting the FA effectively. I had to convincingly sell the idea of the product and explain how it could provide more value than the current solutions. Overcame this by iterating through feedback and refining the analysis.

Outcomes and Impact: The FA was ultimately shelved as it wasn't prioritized, but it provided a detailed plan that could be revisited in the future. It demonstrated the potential for expanding the product's use across the entire group of companies.
